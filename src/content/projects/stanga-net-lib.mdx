## Overview

A suite of 10 independent, composable libraries for .NET 8 and .NET 9, developed with a strict Test-Driven Development approach. Each library has a focused responsibility, zero coupling to the others, and full unit + integration test coverage.

The goal is to encode hard-won architectural decisions into reusable, well-tested building blocks — so they don't need to be reinvented on every new project, and so that the assumptions behind them are explicit and verifiable.

All libraries are published as open-source repositories under the [StangaNet GitHub organisation](https://github.com/StangaNet).

## Library suite

The ecosystem covers 10 foundational libraries. Every library depends on **StangaNetLib.Core**, which provides the shared abstractions, result primitives, and interfaces that keep all other libraries decoupled from infrastructure.

| Library | Responsibility |
|---|---|
| **StangaNetLib.Core** | Foundation library: `Result<T>` with functional combinators (`Map`, `Bind`, `Match`), generic `Entity<TId>` with domain events and audit timestamps, `Guard.Against` clauses, Specification pattern, pagination, and `IUnitOfWork`. No external dependencies. |
| **StangaNetLib.Concurrency** | Keyed async locks, semaphore-based throttle, bounded channel work queues, debouncer, and atomic counters — zero-allocation concurrency primitives built on `System.Threading.Channels` |
| **StangaNetLib.Caching** | Unified caching abstraction over in-memory and Redis; swap-transparent at the composition root |
| **StangaNetLib.Observability** | Structured logging with GDPR-safe automatic field masking — PII fields are redacted before they reach any sink |
| **StangaNetLib.Media** | Image optimisation and multi-provider media storage (local filesystem, Azure Blob Storage) behind a unified interface |
| **StangaNetLib.Auth** | JWT authentication (HMAC-SHA256 self-signed or Auth0 RS256/JWKS), per-device refresh token rotation, BCrypt password hashing, cache-backed token revocation blacklist, and Auth0 Management API integration |
| **StangaNetLib.RateLimit** | Distributed rate limiting middleware with configurable policies per endpoint, user, or IP |
| **StangaNetLib.Idempotency** | Request idempotency middleware — clients can safely retry operations without risk of duplicate side effects |
| **StangaNetLib.Gdpr** | AES-256 pseudonymisation, audit trail, data export, and right-to-erasure flows compliant with GDPR requirements |
| **StangaNetLib.ContentFlow** | Generic multi-step content approval workflow engine with configurable states, transitions, and reviewer assignment |

## Design principles

- **StangaNetLib.Core is the only mandatory dependency** — every other library depends on Core and nothing else in the ecosystem, preserving composability
- **Zero external dependencies at the core** — infrastructure concerns (databases, caches, cloud providers) are kept behind interfaces defined in Core; concrete adapters live in separate packages
- **Composable** — libraries are independent of each other; a project can adopt one, several, or all without pulling in unwanted dependencies
- **TDD throughout** — every public API is specified by tests before the implementation is written; tests document intent, not just correctness
- **Clean Architecture compliant** — all abstractions live in the domain/application layer; infrastructure adapters are in separate assemblies and are never referenced directly by domain code
