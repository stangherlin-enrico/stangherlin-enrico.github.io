## Overview

The site you're reading right now. Built from scratch as a personal space to document projects, write about software, and maintain a professional presence. Every part of the stack was chosen deliberately — there is no framework overhead, no CMS, and no third-party hosting costs.

Content lives as MDX files in the repository, the build is a single Vite pass, and deployment is a GitHub Actions workflow triggered on every push to `main`.

## Architecture

The site follows a deliberate **content-first** architecture:

- **Routing** — React Router v6 with two layout modes: a full-bleed `HomeLayout` for the landing page and a constrained `PageLayout` (max-w-5xl) for all inner pages.
- **Content** — Blog posts and project pages are authored in MDX and resolved at build time via `import.meta.glob`. Frontmatter is exported as typed props by `remark-mdx-frontmatter`.
- **Search** — Client-side full-text search using Fuse.js, indexed on title, excerpt and tags with a fuzzy threshold of 0.35.
- **SEO** — Per-page `<title>`, `<meta description>`, Open Graph and Twitter Card tags injected via `react-helmet-async`.
- **CV generation** — A separate Node script (`scripts/generate-cv.mjs`) uses Puppeteer to render an HTML template and export colour and print-friendly PDF versions. The download button is driven by a `manifest.json` so adding or removing PDF versions requires no code changes.

## Stack

- **React 18 + Vite 6** — fast dev server and tree-shaken production build
- **TypeScript** — strict mode, no unused locals or parameters
- **Tailwind CSS v3** — utility-first styling; `@tailwindcss/typography` for prose; CSS custom properties for the full design token system
- **Syne + Outfit + JetBrains Mono** — display, body and mono fonts loaded via Google Fonts
- **MDX** — blog posts and project pages with embedded React support
- **Framer Motion** — declarative animations (home hero, future transitions)
- **React Router v6** — client-side routing with a `404.html` fallback for GitHub Pages SPA support
- **Fuse.js** — lightweight client-side full-text search
- **react-helmet-async** — per-page SEO meta tags
- **Puppeteer Core** — headless CV PDF generation

## Features

### Portfolio
- Per-project detail pages loaded from MDX with typed frontmatter
- Multi-select tag filtering with AND logic — selecting multiple tags narrows results
- Status badges (Released / In Progress / Archived) and visibility badges (Open Source / Confidential) with coloured borders and tinted backgrounds
- Per-project Changelog section with collapsible UI and type-coded entries

### Blog
- MDX posts with syntax-highlighted code blocks (rehype-highlight)
- Full-text fuzzy search across title, excerpt and tags
- Tag display on each post card

### CV
- Interactive timeline for work experience and education
- Skills grouped by hard/soft categories
- Languages table with CEFR levels
- Certifications grid with credential links
- PDF download dropdown driven by `public/cv/manifest.json` — supports multiple versions (colour, B&W)
- Reference section with Italian/English toggle

### Infrastructure
- GitHub Actions CI/CD — every push to `main` builds, copies `index.html → 404.html` for SPA routing, and deploys to GitHub Pages
- No backend, no database, no hosting costs
